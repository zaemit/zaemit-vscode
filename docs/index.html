Zaemit Visual Editor

VS Code 안에서 HTML/CSS를 시각적으로 편집하세요. 브라우저 없이, 코드 에디터를 떠나지 않고.

VS Code Extension v0.0.5

Installation

VS Code Marketplace에서 설치

  1. VS Code를 엽니다.
    좌측 사이드바에서 확장 프로그램 아이콘(블록 모양)을 클릭하거나, Ctrl+Shift+X를 누릅니다.
  2. 검색창에 "Zaemit"을 입력합니다.
    검색 결과에서 Zaemit Visual Editor를 찾습니다.
  3. Install 버튼을 클릭합니다.
    설치가 완료되면 별도의 설정 없이 바로 사용할 수 있습니다.
IMAGE: marketplace-install.png VS Code Marketplace에서 Zaemit 검색 및 설치 화면

VSIX 파일로 직접 설치

.vsix 파일을 직접 받은 경우, 다음 방법으로 설치할 수 있습니다:

code --install-extension zaemit-visual-editor-0.0.5.vsix --force

또는 VS Code에서 Ctrl+Shift+P → Extensions: Install from VSIX...를 선택하고 파일을 지정합니다.

Opening Your First File

Zaemit Visual Editor로 HTML 파일을 여는 방법은 두 가지입니다:

방법 1: 우클릭 메뉴

  1. VS Code 탐색기에서 .html 파일을 찾습니다.
    프로젝트 내 어떤 HTML 파일이든 가능합니다.
  2. 파일을 마우스 오른쪽 버튼으로 클릭합니다.
    컨텍스트 메뉴가 나타납니다.
  3. "Open with Zaemit Visual Editor"를 선택합니다.
    새 에디터 탭에서 비주얼 에디터가 열립니다.
IMAGE: open-context-menu.png HTML 파일 우클릭 시 "Open with Zaemit Visual Editor" 메뉴

방법 2: 상태바 버튼

VS Code 하단 상태바에 있는 Zaemit 버튼을 클릭하면, 현재 열려 있는 HTML 파일을 비주얼 에디터로 엽니다.

IMAGE: status-bar-button.png VS Code 하단 상태바의 Zaemit 버튼
TIP
HTML 파일이 아닌 파일이 열려 있을 때 Zaemit 버튼을 누르면 "HTML 파일을 열어주세요" 안내가 나타납니다.

Editor Overview

Zaemit Visual Editor는 크게 4개의 영역으로 구성됩니다:

IMAGE: editor-overview-annotated.png 에디터 전체 화면 구성 (각 영역 번호 표시)

1. Canvas (중앙)

HTML 페이지의 라이브 프리뷰. 여기서 요소를 클릭하고, 드래그하고, 수정합니다. iframe 기반으로 실제 브라우저 렌더링과 동일합니다.

2. Layer Panel (좌측)

페이지의 DOM 트리를 시각적으로 보여줍니다. 요소를 탐색하고, 이름을 변경하고, 순서를 바꿀 수 있습니다.

3. Style Panel (우측)

선택한 요소의 CSS 속성을 편집합니다. 레이아웃, 크기, 타이포그래피, 테두리, 배경, 효과 등 모든 스타일을 제어합니다.

4. Toolbar (상단)

뷰 모드 전환(Desktop/Tablet/Mobile), 줌 컨트롤, 저장, 실행취소/다시실행 등의 도구 모음입니다.

Selecting Elements

캔버스에서 요소를 클릭하면 해당 요소가 선택됩니다. 선택된 요소에는 다음이 표시됩니다:

  • 파란색 테두리 — 선택된 요소의 경계를 나타냅니다
  • 리사이즈 핸들 — 8개 방향(모서리 4개 + 변 4개)의 크기 조절 핸들
  • 회전 핸들 — 상단에 회전을 위한 원형 핸들
  • 요소 라벨 — 좌측 상단에 태그명(예: div, h1, img)이 표시됩니다
  • 스페이싱 핸들 — 요소 주변에 마진/패딩 조절 핸들이 나타납니다
IMAGE: element-selected.png 요소 선택 시 나타나는 오버레이 (테두리, 핸들, 라벨 표시)
INFO
캔버스의 빈 영역을 클릭하면 선택이 해제됩니다. Esc 키를 눌러도 선택 해제됩니다.

요소 선택 방법들

  • 캔버스 클릭 — 가장 기본적인 방법. 원하는 요소를 직접 클릭
  • Layer Panel 클릭 — 좌측 레이어 패널에서 요소 이름을 클릭
  • 중첩 요소 — 요소가 겹쳐있는 경우, 라벨의 왼쪽 화살표(◀)를 클릭하면 부모 요소를 선택할 수 있습니다

Moving Elements

선택한 요소를 드래그하면 위치를 이동할 수 있습니다.

드래그 이동

  1. 요소를 클릭하여 선택합니다
  2. 선택된 요소를 마우스로 누른 채 드래그합니다
  3. 원하는 위치에서 마우스를 놓습니다
TIP
CSS position 속성이 static인 요소는 드래그해도 DOM 순서만 변경됩니다. 자유로운 위치 이동을 원하면 Style Panel에서 position을 absolute 또는 relative로 변경하세요.

키보드로 DOM 순서 이동

  • Alt+↑ — 요소를 DOM 트리에서 위로 이동 (형제 요소와 순서 교환)
  • Alt+↓ — 요소를 DOM 트리에서 아래로 이동

Resizing Elements

선택한 요소의 8개 핸들을 드래그하여 크기를 조절합니다:

  • 모서리 핸들 (4개) — 너비와 높이를 동시에 조절
  • 상/하 변 핸들 (2개) — 높이만 조절
  • 좌/우 변 핸들 (2개) — 너비만 조절
IMAGE: resize-handles.png 요소의 8개 리사이즈 핸들 (모서리 + 변)
TIP: 크기 초기화
리사이즈 핸들을 더블클릭하면 크기가 auto로 리셋됩니다. 모서리 핸들을 더블클릭하면 너비와 높이가 동시에 리셋됩니다.

Rotating Elements

선택한 요소 상단의 원형 핸들을 드래그하면 요소를 회전할 수 있습니다.

  • 회전 핸들을 클릭한 채 시계 방향 또는 반시계 방향으로 드래그합니다
  • 회전 각도는 CSS transform: rotate()로 적용됩니다
  • Style Panel의 Effects 섹션에서 정확한 각도를 숫자로 입력할 수도 있습니다

Multi-Select

Ctrl 키를 누른 채 여러 요소를 클릭하면 다중 선택이 가능합니다.

다중 선택으로 할 수 있는 작업

  • 일괄 스타일 변경 — 선택된 모든 요소에 동일한 스타일을 한번에 적용
  • 일괄 이동 — 선택된 요소들을 함께 드래그
  • 일괄 삭제 — Delete 키로 선택된 모든 요소 삭제
  • 정렬 — Alt+L/M/R로 좌/중앙/우 정렬
INFO
Layer Panel에서도 Ctrl+클릭으로 다중 선택이 가능합니다.

Text Editing

텍스트 요소를 더블클릭하면 텍스트 편집 모드에 진입합니다.

텍스트 편집 모드에서 가능한 작업

  • 직접 텍스트 입력 및 수정
  • 텍스트 드래그 선택 → 서식 툴바 표시
  • Ctrl+C/V — 텍스트 복사/붙여넣기
  • Ctrl+Z — 텍스트 편집 되돌리기
  • Enter — 줄바꿈 (<br> 삽입)
  • Esc — 텍스트 편집 모드 종료
IMAGE: text-editing-toolbar.png 텍스트 선택 시 나타나는 인라인 서식 툴바 (Bold, Italic 등)

텍스트 서식 단축키

기능단축키

밑줄 (Underline)Alt+Shift+U
글꼴 크기 증가Alt+Shift+=

Drag & Drop Reorder

Layer Panel이나 캔버스에서 요소를 드래그하여 DOM 트리의 순서를 변경할 수 있습니다.

  • Layer Panel에서 요소를 다른 위치로 드래그하면 부모/형제 관계가 변경됩니다
  • 캔버스에서 드래그 시, Flex/Grid 컨테이너의 빈 셀에 드롭하면 해당 위치에 요소가 삽입됩니다
  • 드래그 중 파란색 가이드 라인이 삽입 위치를 보여줍니다

Style Panel Overview

우측 Style Panel은 선택한 요소의 모든 CSS 속성을 시각적으로 편집할 수 있는 강력한 도구입니다. 7개의 섹션으로 나뉘어 있습니다.

IMAGE: style-panel-full.png Style Panel 전체 모습 (섹션별 접기/펼치기 상태)
INFO: CSS 규칙과 인라인 스타일
Zaemit은 가능하면 CSS 규칙(stylesheet)에 스타일을 적용합니다. 인라인 스타일이 이미 있는 경우 인라인을 수정하고, CSS 규칙이 있는 경우 가장 높은 specificity의 규칙을 수정합니다. 이렇게 하면 기존 CSS 구조를 깨뜨리지 않습니다.

Layout

요소의 레이아웃 관련 속성을 설정합니다.

Display

요소의 표시 방식을 선택합니다:

  • block — 블록 레벨 요소 (전체 너비 차지)
  • inline — 인라인 요소 (콘텐츠 너비만 차지)
  • inline-block — 인라인이지만 블록 속성 사용 가능
  • flex — Flexbox 레이아웃 컨테이너
  • grid — Grid 레이아웃 컨테이너
  • none — 요소를 숨김

Flexbox 속성

Display가 flex일 때 추가 속성이 나타납니다:

  • Direction — row, column, row-reverse, column-reverse
  • Wrap — nowrap, wrap, wrap-reverse
  • Justify Content — 주축 방향 정렬 (start, center, end, space-between, space-around, space-evenly)
  • Align Items — 교차축 방향 정렬 (start, center, end, stretch, baseline)
  • Gap — 자식 요소 간 간격
IMAGE: style-layout-flex.png Flexbox 레이아웃 설정 UI (Direction, Justify, Align 버튼 그룹)

Position

요소의 위치 지정 방식을 선택합니다:

  • static — 기본값, 문서 흐름에 따름
  • relative — 원래 위치에서 상대적으로 이동
  • absolute — 가장 가까운 positioned 부모 기준으로 절대 위치
  • fixed — 뷰포트 기준으로 고정
  • sticky — 스크롤 시 고정 위치에 붙음

Position이 static이 아닌 경우, top, right, bottom, left 값을 입력할 수 있습니다.

Z-Index

요소의 쌓임 순서를 제어합니다. 숫자가 높을수록 위에 표시됩니다.

  • Alt+Shift+↑ — z-index 증가
  • Alt+Shift+↓ — z-index 감소

Size & Spacing

요소의 크기(너비, 높이)와 간격(마진, 패딩)을 설정합니다.

Width / Height

  • 숫자 입력 (px, %, em, rem, vw, vh 등 단위 지원)
  • auto — 콘텐츠에 맞게 자동 계산
  • fit-content — 콘텐츠에 맞춤
  • 100% — 부모 요소에 맞춤

Min/Max Width & Height

최소/최대 크기를 설정하여 반응형 디자인에 활용합니다.

Margin (바깥 여백)

요소 바깥의 여백입니다. 상/우/하/좌 각각 개별 설정 가능합니다.

TIP: 스페이싱 드래그
캔버스에서 요소를 선택하면, 요소 주변에 연보라색 스페이싱 핸들이 나타납니다. 이 핸들을 드래그하면 마진과 패딩을 시각적으로 조절할 수 있습니다.
IMAGE: spacing-drag-handles.png 마진/패딩 스페이싱 핸들 드래그 조절 화면

Padding (안쪽 여백)

요소 안쪽의 여백입니다. 콘텐츠와 테두리 사이의 공간을 제어합니다.

Typography

텍스트 관련 모든 CSS 속성을 편집합니다.

Font Family

글꼴을 선택합니다. 시스템 폰트 및 웹폰트(Google Fonts 등)를 지원합니다.

Font Size

글꼴 크기를 설정합니다. px, em, rem 등 다양한 단위를 지원합니다.

Font Weight

글꼴 두께를 선택합니다:

  • 100 (Thin) ~ 900 (Black)
  • normal (400), bold (700)

Text Alignment

텍스트 정렬 방식을 선택합니다:

  • Alt+Shift+L — 좌측 정렬
  • Alt+Shift+E — 중앙 정렬
  • Alt+Shift+R — 우측 정렬
  • Alt+Shift+J — 양쪽 정렬

기타 속성

  • Line Height — 줄 간격
  • Letter Spacing — 글자 간격
  • Text Decoration — 밑줄, 윗줄, 취소선
  • Text Transform — 대문자, 소문자, 첫 글자 대문자
  • Color — 텍스트 색상 (컬러 피커 제공)

Border

요소의 테두리를 설정합니다. 개별 면(상/우/하/좌)을 각각 다르게 설정할 수 있습니다.

IMAGE: style-border-panel.png Border 섹션 UI (면 선택, 스타일/두께/색상/반경 설정)

Border 속성

  • Width — 테두리 두께 (px)
  • Style — solid, dashed, dotted, double, none 등
  • Color — 테두리 색상 (컬러 피커)

Border Radius

모서리 둥글기를 설정합니다. 4개 모서리를 각각 다르게 설정할 수 있습니다.

  • 숫자 입력 (px, %)
  • 50%로 설정하면 원형이 됩니다

Background

요소의 배경을 설정합니다.

Background Color

배경 색상을 설정합니다. 컬러 피커에서 직접 선택하거나, HEX/RGB/HSL 값을 입력합니다.

Background Image

  • URL 입력 — 이미지 경로를 직접 입력
  • Size — cover, contain, auto 또는 직접 입력
  • Position — center, top, bottom 등
  • Repeat — repeat, no-repeat, repeat-x, repeat-y

Effects

시각 효과 관련 속성을 설정합니다.

Opacity

요소의 투명도를 0(완전 투명) ~ 1(완전 불투명) 사이로 설정합니다.

Box Shadow

요소에 그림자를 추가합니다:

  • Offset X/Y — 그림자 위치
  • Blur — 흐림 정도
  • Spread — 그림자 확산
  • Color — 그림자 색상
  • Inset — 내부 그림자 여부

Transform

  • Rotate — 회전 각도 (deg)
  • Scale — 크기 배율
  • Translate — X/Y 이동
  • Skew — 기울이기

Transition

CSS 속성 변화에 애니메이션을 추가합니다:

  • Property — 애니메이션할 CSS 속성 (all, opacity, transform 등)
  • Duration — 지속 시간 (초)
  • Timing — ease, linear, ease-in, ease-out, ease-in-out

Cursor

마우스 커서 모양을 변경합니다: pointer, crosshair, grab, text 등

Overflow

요소의 콘텐츠가 크기를 초과할 때의 동작을 설정합니다.

  • visible — 넘치는 콘텐츠를 그대로 표시
  • hidden — 넘치는 콘텐츠를 숨김
  • scroll — 항상 스크롤바 표시
  • auto — 필요할 때만 스크롤바 표시

X축(수평)과 Y축(수직)을 개별적으로 설정할 수 있습니다.

Pseudo-Class States

Zaemit은 요소의 다양한 상태에 대한 스타일을 편집할 수 있습니다.

지원하는 상태

  • Default — 기본 상태
  • :hover — 마우스를 올렸을 때
  • :focus — 요소에 포커스가 갔을 때
  • :active — 클릭 중일 때
IMAGE: pseudo-class-selector.png Style Panel 상단의 상태 선택 UI (Default / :hover / :focus / :active)
INFO
Pseudo-class 스타일은 인라인 스타일이 아닌 CSS 규칙(stylesheet)으로 저장됩니다. CSS에서 pseudo-class는 인라인으로 적용할 수 없기 때문입니다.

사용 예시

  2. Style Panel 상단에서 :hover를 선택합니다
  3. 배경 색상을 더 밝은 색으로 변경합니다
  4. 이제 실제 브라우저에서 마우스를 올리면 색상이 변합니다

Undo / Redo

Zaemit은 모든 작업을 기록하며, 언제든 되돌리거나 다시 실행할 수 있습니다.

  • Ctrl+Z — 실행 취소 (Undo)
  • Ctrl+Y 또는 Ctrl+Shift+Z — 다시 실행 (Redo)

기록되는 변경 사항

  • CSS 스타일 변경
  • 요소 이동, 크기 변경, 회전
  • 텍스트 콘텐츠 수정
  • 요소 추가, 삭제, 순서 변경
  • HTML 속성 변경
  • 태그 변경
INFO
텍스트 편집 중에 Ctrl+Z를 누르면 텍스트 편집 범위 내에서만 되돌립니다. 텍스트 편집을 종료한 후 Ctrl+Z를 누르면 이전 작업(요소 이동, 스타일 변경 등)을 되돌립니다.

FAQ

Q: 어떤 파일 형식을 지원하나요?

.html 파일만 지원합니다. CSS와 JavaScript는 HTML 파일과 함께 자동으로 로드됩니다.

Q: 기존 프로젝트에서도 사용할 수 있나요?

네, 어떤 HTML 파일이든 열 수 있습니다. 기존 CSS 규칙을 인식하고 수정하므로, 프로젝트의 기존 구조를 깨뜨리지 않습니다.

Q: 변경사항은 어디에 저장되나요?

원본 HTML 파일에 직접 저장됩니다. CSS 스타일 변경은 HTML 내의 <style> 태그 또는 연결된 CSS 파일에 적용됩니다.

Q: 외부 CSS 프레임워크(Bootstrap, Tailwind)와 호환되나요?

네, 외부 CSS를 사용하는 페이지에서도 정상 작동합니다. Zaemit은 기존 CSS 규칙의 specificity를 인식하고, 가장 적절한 위치에 스타일을 적용합니다.

Q: 여러 HTML 파일을 동시에 편집할 수 있나요?

네, 여러 HTML 파일을 각각 다른 탭에서 열어 편집할 수 있습니다.

Q: 자동 저장이 되나요?

스타일이나 요소를 변경하면 자동으로 파일에 반영됩니다. VS Code의 파일 저장(Ctrl+S)을 별도로 할 필요가 없습니다.

Troubleshooting

에디터가 빈 화면으로 표시됩니다

  • HTML 파일의 문법이 올바른지 확인하세요
  • VS Code를 재시작해보세요
  • 확장 프로그램을 재설치해보세요

스타일 변경이 적용되지 않습니다

  • 다른 CSS 규칙이 더 높은 specificity로 스타일을 덮어쓸 수 있습니다
  • !important가 사용된 규칙은 우선순위가 높아 변경이 어려울 수 있습니다
  • Code Editor 탭에서 직접 CSS를 확인해보세요

외부 리소스(이미지, 폰트)가 표시되지 않습니다

  • VS Code WebView의 보안 정책(CSP)으로 인해, 일부 외부 URL이 차단될 수 있습니다
  • 가능하면 로컬 파일을 사용하세요
  • CDN URL(예: Google Fonts, cdnjs)은 대부분 허용됩니다

캔버스가 응답하지 않습니다

  • JavaScript 에러로 인해 페이지가 멈출 수 있습니다
  • VS Code 개발자 도구(Ctrl+Shift+I)에서 콘솔 에러를 확인하세요
  • HTML 파일을 다시 열어보세요
