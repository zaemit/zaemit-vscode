Zaemit Visual Editor

Edit HTML/CSS visually inside VS Code. No browser needed, no context switching.

VS Code Extension v0.0.7
WHAT'S NEW IN v0.0.7
  • CSS Undo/Redo fix — Background-image removal and other CSS rule changes now correctly undo with Ctrl+Z
  • Single-view editing — CSS edits in single-view mode no longer propagate to other breakpoints
  • Gap drag sync — Fixed gap shorthand resolution and inline splitting for multiview

Installation

Install from VS Code Marketplace

  1. Open VS Code.
    Click the Extensions icon in the left sidebar, or press Ctrl+Shift+X.
  2. Search for "Zaemit" in the search bar.
    Find Zaemit Visual Editor in the search results.
  3. Click the Install button.
    Once installed, no additional configuration is needed. You're ready to go.
IMAGE: marketplace-install.png VS Code Marketplace search results showing Zaemit Visual Editor

Install from VSIX File

If you received a .vsix file directly, you can install it using the command line:

code --install-extension zaemit-visual-editor-0.0.7.vsix --force

Alternatively, open the Command Palette (Ctrl+Shift+P) and select Extensions: Install from VSIX..., then choose the file.

Opening Your First File

There are two ways to open an HTML file with Zaemit Visual Editor:

Method 1: Right-Click Context Menu

  1. Find any .html file in the VS Code Explorer.
    Any HTML file in your project will work.
  2. Right-click on the file.
    A context menu will appear.
  3. Select "Open with Zaemit Visual Editor".
    The visual editor opens in a new editor tab.
IMAGE: open-context-menu.png Right-click context menu showing "Open with Zaemit Visual Editor" option

Method 2: Status Bar Button

Click the Zaemit button in the bottom status bar of VS Code. This opens the currently active HTML file in the visual editor.

IMAGE: status-bar-button.png VS Code status bar with the Zaemit button highlighted
TIP
If a non-HTML file is currently open, clicking the Zaemit button will show a message asking you to open an HTML file first.

Editor Overview

The Zaemit Visual Editor interface consists of four main areas:

IMAGE: editor-overview-annotated.png Full editor layout with numbered annotations for each area

1. Canvas (Center)

A live preview of your HTML page. Click, drag, and modify elements directly here. Rendered in an iframe for accurate browser-like rendering.

2. Layer Panel (Left)

Displays the DOM tree visually. Browse, select, rename, and reorder elements in the hierarchy.

3. Style Panel (Right)

Edit CSS properties of the selected element. Controls for layout, size, typography, borders, backgrounds, effects, and more.

4. Toolbar (Top)

View mode switching (Desktop/Tablet/Mobile), zoom controls, save, undo/redo, and other tools.

Selecting Elements

Click on any element in the canvas to select it. When selected, the following indicators appear:

  • Blue border — Outlines the boundaries of the selected element
  • Resize handles — 8 handles (4 corners + 4 edges) for resizing
  • Rotation handle — A circular handle above the element for rotation
  • Element label — Tag name (e.g., div, h1, img) shown at the top-left corner
  • Spacing handles — Handles around the element for adjusting margin and padding
IMAGE: element-selected.png Selected element with overlay showing border, handles, and label
INFO
Click on an empty area of the canvas to deselect. You can also press Esc to deselect.

Ways to Select Elements

  • Canvas click — The most common method. Click directly on the desired element.
  • Layer Panel click — Click an element name in the left Layer Panel.
  • Nested elements — If elements overlap, click the left arrow (◀) on the label to select the parent element.

Moving Elements

Drag a selected element to move it to a new position.

Drag to Move

  1. Click an element to select it.
  2. Press and hold the mouse button, then drag.
  3. Release the mouse at the desired location.
TIP
Elements with CSS position: static (the default) will change their DOM order when dragged, not their pixel position. To freely position an element, change its position to absolute or relative in the Style Panel.

Reorder with Keyboard

  • Alt+↑ — Move element up in the DOM tree (swap with previous sibling)
  • Alt+↓ — Move element down in the DOM tree (swap with next sibling)

Resizing Elements

Drag any of the 8 handles on a selected element to resize it:

  • Corner handles (4) — Resize both width and height simultaneously
  • Top/bottom edge handles (2) — Resize height only
  • Left/right edge handles (2) — Resize width only
IMAGE: resize-handles.png Element showing all 8 resize handles (corners and edges)
TIP: Reset Size
Double-click a resize handle to reset the dimension to auto. Double-clicking a corner handle resets both width and height at once.

Rotating Elements

Drag the circular handle above the selected element to rotate it.

  • Click and drag the rotation handle clockwise or counter-clockwise.
  • The rotation is applied as CSS transform: rotate().
  • You can also enter an exact angle in the Style Panel's Effects section.

Multi-Select

Hold Ctrl and click multiple elements to select them all at once.

What You Can Do with Multi-Select

  • Batch style changes — Apply the same style to all selected elements at once
  • Batch move — Drag selected elements together
  • Batch delete — Press Delete to remove all selected elements
  • Alignment — Alt+L/M/R to align left/center/right
INFO
Multi-select also works in the Layer Panel — Ctrl+click to select multiple items.

Text Editing

Double-click a text element to enter text editing mode.

Available Actions in Text Editing Mode

  • Type directly to modify text content
  • Select text with the mouse to reveal the formatting toolbar
  • Ctrl+C/V — Copy/paste text
  • Ctrl+Z — Undo text changes
  • Enter — Insert a line break (<br>)
  • Esc — Exit text editing mode

Drag & Drop Reorder

You can reorder elements in the DOM tree by dragging them in the Layer Panel or on the canvas.

  • Layer Panel — Drag an element to a new position to change its parent or sibling order.
  • Canvas — When dragging over a Flex/Grid container, drop into empty cells to insert at that position.
  • A blue guide line shows the insertion point during drag.

Style Panel Overview

The Style Panel on the right side lets you visually edit all CSS properties of the selected element. It's organized into 7 collapsible sections.

IMAGE: style-panel-full.png Full Style Panel with all sections (some collapsed, some expanded)
INFO: CSS Rules vs. Inline Styles
Zaemit intelligently applies styles to the appropriate location. If the element already has inline styles, those are modified. If CSS rules exist in a stylesheet, Zaemit finds the rule with the highest specificity and modifies it there. This preserves your existing CSS structure.

Layout

Configure how elements are positioned and laid out on the page.

Display

Choose how the element is rendered:

  • block — Block-level element (takes full width)
  • inline — Inline element (takes only content width)
  • inline-block — Inline but supports block properties
  • flex — Flexbox layout container
  • grid — Grid layout container
  • none — Hides the element

Flexbox Properties

When display is set to flex, additional controls appear:

  • Direction — row, column, row-reverse, column-reverse
  • Wrap — nowrap, wrap, wrap-reverse
  • Justify Content — Main-axis alignment (start, center, end, space-between, space-around, space-evenly)
  • Align Items — Cross-axis alignment (start, center, end, stretch, baseline)
  • Gap — Spacing between child elements
IMAGE: style-layout-flex.png Flexbox layout controls (Direction, Justify, Align button groups)

Position

Choose how the element is positioned:

  • static — Default, follows normal document flow
  • relative — Offset from its original position
  • absolute — Positioned relative to the nearest positioned ancestor
  • fixed — Fixed relative to the viewport
  • sticky — Sticks to a position when scrolling

When position is not static, inputs for top, right, bottom, left become available.

Z-Index

Controls the stacking order of elements. Higher values appear on top.

  • Alt+Shift+↑ — Increase z-index
  • Alt+Shift+↓ — Decrease z-index

Size & Spacing

Control element dimensions (width, height) and spacing (margin, padding).

Width / Height

  • Enter numeric values with units (px, %, em, rem, vw, vh, etc.)
  • auto — Let the browser calculate based on content
  • fit-content — Shrink to fit content
  • 100% — Match parent width/height

Min/Max Width & Height

Set minimum and maximum size constraints for responsive design.

Margin (Outer Spacing)

Space outside the element. Each side (top, right, bottom, left) can be set independently.

TIP: Visual Spacing Drag
When an element is selected on the canvas, light-purple spacing handles appear around it. Drag these handles to visually adjust margin and padding without using the Style Panel.
IMAGE: spacing-drag-handles.png Margin/padding spacing handles being dragged on the canvas

Padding (Inner Spacing)

Space inside the element, between the content and the border. Each side can be set independently.

Typography

Edit all text-related CSS properties.

Font Family

Select the typeface. System fonts and web fonts (e.g., Google Fonts loaded via CDN) are supported.

Font Size

Set the text size. Supports px, em, rem, and other CSS units.

Font Weight

Choose the thickness of the font:

  • 100 (Thin) through 900 (Black)
  • normal (400), bold (700)

Text Alignment

Choose how text is aligned within the element:

  • Alt+Shift+L — Align left
  • Alt+Shift+E — Align center
  • Alt+Shift+R — Align right
  • Alt+Shift+J — Justify

Additional Properties

  • Line Height — Spacing between lines of text
  • Letter Spacing — Spacing between characters
  • Text Decoration — Underline, overline, line-through
  • Text Transform — Uppercase, lowercase, capitalize
  • Color — Text color (with color picker)

Border

Configure element borders. You can set each side (top, right, bottom, left) independently.

IMAGE: style-border-panel.png Border section UI showing side selection, style/width/color/radius controls

Border Properties

  • Width — Border thickness in pixels
  • Style — solid, dashed, dotted, double, none, etc.
  • Color — Border color (with color picker)

Border Radius

Control corner roundness. Each of the 4 corners can be set independently.

  • Enter numeric values (px, %)
  • Set to 50% to create a circle

Background

Configure the element's background.

Background Color

Set the background color using the color picker, or enter HEX/RGB/HSL values directly.

Background Image

  • URL — Enter the image path directly
  • Size — cover, contain, auto, or custom values
  • Position — center, top, bottom, left, right, etc.
  • Repeat — repeat, no-repeat, repeat-x, repeat-y

Effects

Configure visual effect properties.

Opacity

Set the element's transparency from 0 (fully transparent) to 1 (fully opaque).

Box Shadow

Add drop shadows to elements:

  • Offset X/Y — Shadow position
  • Blur — Blur radius
  • Spread — Shadow expansion
  • Color — Shadow color
  • Inset — Toggle inner shadow

Transform

  • Rotate — Rotation angle in degrees
  • Scale — Size multiplier
  • Translate — X/Y offset
  • Skew — Tilt angle

Transition

Add animations to CSS property changes:

  • Property — Which CSS property to animate (all, opacity, transform, etc.)
  • Duration — How long the animation lasts (seconds)
  • Timing — ease, linear, ease-in, ease-out, ease-in-out

Cursor

Change the mouse cursor appearance: pointer, crosshair, grab, text, etc.

Overflow

Control what happens when content overflows the element's boundaries.

  • visible — Overflowing content is displayed as-is
  • hidden — Overflowing content is clipped
  • scroll — Always show scrollbars
  • auto — Show scrollbars only when needed

The X-axis (horizontal) and Y-axis (vertical) can be configured independently.

Pseudo-Class States

Zaemit lets you edit styles for different interactive states of an element.

Supported States

  • Default — Normal state
  • :hover — When the mouse is over the element
  • :focus — When the element has keyboard focus
  • :active — When the element is being clicked
IMAGE: pseudo-class-selector.png State selector at the top of the Style Panel (Default / :hover / :focus / :active)
INFO
Pseudo-class styles are saved as CSS rules in the stylesheet, not as inline styles. This is because CSS pseudo-classes cannot be applied inline.

Example Workflow

  1. Select a button element
  2. Choose :hover from the state selector at the top of the Style Panel
  3. Change the background color to a lighter shade
  4. Now when you hover over the button in a browser, the color changes

Layer Panel

The Layer Panel on the left displays the page's DOM tree in a visual hierarchy.

IMAGE: layer-panel.png Layer Panel showing DOM tree with expand/collapse, selection state

Features

  • Browse elements — Expand or collapse tree nodes to find elements
  • Select elements — Click an element name to select it on the canvas
  • Multi-select — Ctrl+click to select multiple elements
  • Rename — Press F2 or double-click to rename an element (sets a data-label attribute)
  • Reorder — Drag elements to move them to a different position in the tree

Toggle

Press Ctrl+Shift+L to show or hide the Layer Panel.

Property Panel

The Property Panel lets you edit HTML attributes of the selected element.

Editable Properties

  • Tag — Change the HTML tag (e.g., div → section)
  • ID — The element's unique identifier
  • Class — CSS class list
  • Custom Attributes — data-*, aria-*, and other custom attributes
  • Link (href) — URL for anchor tags
  • Image (src, alt) — Image source and alternative text for img tags

Code Editor

The built-in Code Editor lets you view and edit the raw HTML/CSS/JS code alongside the visual canvas.

Features

  • Syntax highlighting — Powered by CodeMirror with HTML/CSS/JS support
  • Two-way sync — Code changes instantly update the canvas, and canvas changes update the code
  • File switching — Switch between index.html, style.css, and script.js tabs
IMAGE: code-editor.png Code editor with syntax highlighting and file tabs

Zoom & Pan

Zoom

  • Ctrl+Scroll Up — Zoom in
  • Ctrl+Scroll Down — Zoom out
  • Ctrl+0 — Reset zoom to 100%

Pan (Move the Canvas)

  • Space + mouse drag — Freely move the canvas
  • On trackpads, two-finger scroll also works for panning
TIP
Zaemit automatically distinguishes between PC mouse wheels and trackpad gestures. Pinch-to-zoom is supported on trackpads.

Undo / Redo

Every action is recorded in the change history and can be undone or redone at any time.

  • Ctrl+Z — Undo
  • Ctrl+Y or Ctrl+Shift+Z — Redo

Tracked Changes

  • CSS style modifications
  • Element moves, resizes, and rotations
  • Text content edits
  • Element additions, deletions, and reordering
  • HTML attribute changes
  • Tag changes
INFO
While in text editing mode, Ctrl+Z undoes text changes only. After exiting text editing mode, Ctrl+Z undoes previous actions (element moves, style changes, etc.).

FAQ

Q: What file types are supported?

Only .html files are supported. CSS and JavaScript files referenced by the HTML are loaded automatically.

Q: Can I use it with existing projects?

Yes! You can open any HTML file. Zaemit recognizes and modifies existing CSS rules without breaking your project's CSS structure.

Q: Where are changes saved?

Changes are saved directly to the original HTML file. CSS style changes are applied to <style> tags within the HTML or to linked CSS files.

Q: Does it work with CSS frameworks (Bootstrap, Tailwind)?

Yes. Zaemit respects existing CSS rules and their specificity. It applies your style changes to the most appropriate location without conflicting with framework styles.

Q: Can I edit multiple HTML files at once?

Yes. You can open multiple HTML files in separate editor tabs, each with its own visual editor instance.

Q: Is auto-save supported?

Yes. Style and element changes are automatically reflected in the file. You don't need to manually save with Ctrl+S after each visual edit.

Q: Does it support macOS and Linux?

Yes. Zaemit Visual Editor works on Windows, macOS, and Linux — anywhere VS Code runs.

Troubleshooting

The editor shows a blank screen

  • Make sure the HTML file has valid syntax.
  • Try restarting VS Code.
  • Reinstall the extension.

Style changes are not being applied

  • Another CSS rule with higher specificity might be overriding your change.
  • Rules with !important have the highest priority and are harder to override.
  • Check the Code Editor tab to inspect the CSS directly.

External resources (images, fonts) are not loading

  • VS Code WebView's Content Security Policy (CSP) may block some external URLs.
  • Use local files when possible.
  • Most CDN URLs (e.g., Google Fonts, cdnjs) are allowed by default.

The canvas is unresponsive

  • A JavaScript error in your page might be freezing the iframe.
  • Open VS Code Developer Tools (Ctrl+Shift+I) and check the console for errors.
  • Try reopening the HTML file.

Changes don't sync to the code editor

  • Make sure you're viewing the correct file tab (index.html, style.css, etc.).
  • Try clicking on a different element and back to refresh the panel.
